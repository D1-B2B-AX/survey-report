# [사용 가이드] 만족도 리포팅 플러그인

> 만족도 설문 raw data와 강의관리 시트를 입력하면, **기업담당자용 + 강사용 메일 초안**을 자동으로 생성하는 Claude Code 플러그인입니다.
> 객관식 점수 계산, 주관식 유형 분류, 운영진 의견 초안까지 한 번에 생성하고, HTML 파일로 블록별 검토 + 메일 미리보기를 제공합니다.

---

## 준비물

| 파일 | 필수 여부 | 설명 |
|------|----------|------|
| **만족도 설문 raw data** | 필수 | 엑셀 파일 (.xlsx). 구글 드라이브에만 있는 경우 먼저 다운로드해주세요 |
| **강의관리 시트** | 선택 | 엑셀 파일. 별도 파일이 없어도 싱크업 문서에 강의관리 탭이 있으면 그 싱크업 문서를 전달해도 됩니다. 없으면 운영진 의견이 일반적인 수준으로 생성됩니다 |

두 파일 모두 **PC에 저장된 상태**여야 합니다.

---

## 설치 방법

터미널(또는 Claude Code 대화창)에서 아래 명령어를 입력하세요:

```bash
git clone https://github.com/D1-B2B-AX/survey-report.git ~/.claude/skills/report
```

첫 실행 시 필요한 패키지가 자동으로 설치됩니다.

---

## 권한 설정 (선택)

리포팅 중 명령 실행 승인 팝업이 뜰 수 있습니다. 아래 방법으로 설정하면 일부 승인 절차가 자동으로 진행됩니다.

**방법 1: 팝업에서 바로 설정**
처음 승인 팝업이 뜨면 두 번째 선택지(프로젝트/세션 단위 허용)를 선택

**방법 2: 영구 설정**
`/permissions` 입력 → `Bash("cd ~/.claude/skills/report*")` → Allow 추가

> 설정하지 않아도 리포팅은 정상 동작합니다. 매 단계마다 승인을 눌러야 해서 번거로울 수 있습니다.

## 사용 방법

### STEP 1. 실행 + 파일 전달

Claude Code 대화창에서 `/report`를 입력합니다.

실행하면 먼저 **환경 설정이 자동으로 확인**됩니다 (패키지 설치, 플러그인 업데이트 등). 완료 후 아래 안내가 나옵니다:

```
만족도 결과보고를 작성할게요. 아래 파일을 전달해주세요.
1. 만족도 설문 raw data (필수)
2. 강의관리 시트 (선택 — 있으면 운영진 의견이 더 구체적으로 생성됩니다)

💡 파일 전달 방법:
- 파일을 대화창에 드래그해도 됩니다
- 또는 파일 경로를 직접 입력: 파일 탐색기에서 파일 클릭 → Ctrl+Shift+C → 붙여넣기

📌 다른 과정의 리포팅을 진행하실 때는 새 터미널에서 Claude Code를 열어주세요. (1과정 1세션)
```

안내에 따라 **raw data**(필수)와 **강의관리 시트**(선택)를 전달합니다.

> 별도 강의관리 시트가 없어도, 싱크업 문서에 **강의관리 탭**이 작성되어 있으면 그 싱크업 문서를 전달해도 됩니다.

강의관리 시트가 없으면 **'없음'**이라고 입력해주세요.

### STEP 2. 메타 정보 확인

파일에서 자동 추출된 메타 정보를 확인합니다:

```
아래 메타 정보를 확인해주세요.

- 기업명: HL그룹
- 과정명: 팀장 리더십 AI
- 교육일시: 2026-03-10 ~ 2026-03-11
- 강사: 김태원
- 응답인원: 25명
- 리포트 범위: 전체 (시트 1개)

맞으면 '확인', 수정할 부분이 있으면 알려주세요.
특이사항이 있으면 함께 알려주세요. (예: "강사용만 필요합니다")
```

- 맞으면 **'확인'**, 수정할 부분이 있으면 알려주세요.
- **다중 시트/그룹이 감지된 경우** 어떤 범위로 리포팅할지 함께 물어봅니다. (예: "전체 통합 vs A그룹만", "0221 시트만 vs 전부")
- **특이사항**이 있으면 이 단계에서 함께 알려주세요. (예: "강사용만 필요합니다", "1차수만 리포팅하고 싶습니다")

### STEP 3. 자동 생성

객관식 계산 → 주관식 정리 → 운영진 의견 초안 → 검증까지 순차적으로 진행됩니다. 각 단계 완료 시 진행 상황을 안내해드립니다.

### STEP 4. 결과 확인

완료되면 HTML 파일이 **브라우저에서 자동으로 열립니다**:

```
리포팅이 완료되었습니다. 브라우저에서 결과를 확인해주세요.
- 블록 검토 탭: 객관식/주관식/운영진 의견을 영역별로 확인 + 근거 자료
- 메일 미리보기 탭: 제목 + 본문 확인, "본문 복사" / "전체 복사" 버튼

수정이 필요하면 대화창에서 말씀해주세요. (예: "운영진 의견 톤 좀 더 부드럽게 해주세요")
Gmail 임시보관함에 메일 초안을 만들어드릴 수도 있습니다. (예: "임시보관함에 넣어줘")
```

- **블록 검토 탭**: 객관식/주관식/운영진 의견을 영역별로 확인할 수 있습니다. 근거 자료도 함께 제공됩니다.
- **메일 미리보기 탭**: 메일 제목 + 본문을 확인할 수 있습니다. "본문 복사" / "전체 복사 (제목+본문)" 버튼으로 바로 복사할 수 있습니다.

수정이 필요하면 대화창에서 요청하시면 됩니다. (예: "운영진 의견 톤을 좀 더 부드럽게 해주세요")

Gmail 임시보관함에 메일 초안을 만들어드릴 수도 있습니다. 원하시면 말씀해주세요. (예: "임시보관함에 넣어줘")
- Gmail MCP가 연결되어 있어야 합니다. 연결되어 있지 않으면 복사 버튼을 사용해주세요.
- 초안은 **수신자 없이** 생성됩니다. Gmail에서 수신자를 추가하고, OOO을 본인 이름으로 바꾼 후 전송해주세요.

## 결과물 안내

HTML 파일 1개가 생성됩니다.
- **파일명**: `기업명_과정명_만족도 리포트.html`
- **저장 위치**: 만족도 조사 raw data를 주신 것과 같은 폴더에 저장됩니다.

### 블록 검토 탭

| 영역 | 내용 |
|------|------|
| 객관식 | 문항별 평균 점수 + 긍정 비율 (4~5점 기준) |
| 주관식 | 긍정/개선/추가 교육 니즈 유형 분류 + 근거 원문 |
| 운영진 의견 | 기업담당자용/강사용 초안 + 작성 근거 표 |

### 메일 미리보기 탭

| 탭 | 내용 |
|----|------|
| 기업담당자용 | 메일 제목 + 객관식 표 + 주관식 요약 + 운영진 의견 + 복사 버튼 (본문/전체) |
| 강사용 | 동일 구조 (강사 보호 필터링 적용) + 복사 버튼 (본문/전체) |

## 예상 소요시간

보통 **8~10분 내외**입니다. 응답 규모가 크거나 다중 시트/강사가 있는 경우 더 걸릴 수 있습니다.

---

## 수정 요청 예시

대화창에서 자유롭게 요청하시면 됩니다:

| 상황 | 요청 예시 |
|------|----------|
| 운영진 의견 톤 변경 | "운영진 의견을 좀 더 부드럽게 해주세요" |
| 주관식 유형 수정 | "이 유형은 빼주세요" / "이 응답을 포함시켜주세요" |
| 객관식 표 수정 | "긍정 비율을 빼주세요" / "순서를 바꿔주세요" |
| 메일 문구 수정 | "마무리 멘트를 바꿔주세요" |
| 추가 정보 활용 | "수강자 소속 분포도 넣어주세요" |

수정 후에는 검증 → HTML 재생성까지 자동으로 다시 진행됩니다.

## 참고 사항

- **객관식 점수는 코드가 계산합니다**
- **주관식은 원문 기반입니다** — AI가 없는 내용을 지어내지 않습니다. 블록 검토에서 근거 원문을 확인할 수 있습니다
- **운영진 의견만 AI가 초안을 작성합니다** — 강의관리시트의 현장 관찰을 바탕으로 작성됩니다. 근거 표에서 확인 후 수정 요청이 가능합니다
- **다른 과정의 리포팅을 진행하실 때는 새 터미널에서 Claude Code를 열어주세요** (1과정 1세션) — 한 세션에서 여러 과정을 처리하면 세션 로그가 길어져 피드백 시 내용 추적이 어려워집니다

## 피드백 / 문의

사용 중 불편한 점이나 개선 의견이 있으시면 아래 방법으로 전달해주세요.

### 피드백 보내는 방법

1. Claude Code 대화창에 **'세션 로그 추출'**이라고 입력하면 md 파일이 생성됩니다
2. 아래 파일을 **AX팀**에게 함께 전달해주세요:
   - 추출된 세션 로그 (.md)
   - 사용한 raw data 파일 (.xlsx)
   - 사용한 강의관리 시트 (.xlsx, 있는 경우)
3. 데이터가 함께 있어야 정확한 원인 파악과 개선이 가능합니다

플러그인 관련 문의, 개선 요청, 버그 리포트 모두 환영합니다
